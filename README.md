# ☕ EspressoPOS - Coffee & Bakery POS System

ระบบจัดการการขายหน้าร้าน (POS) สไตล์ Light Mode สำหรับร้านกาแฟและเบเกอรี่ พร้อมระบบสะสมแต้มสมาชิก เชื่อมโยงฐานข้อมูล Supabase และรายงานรายงานยอดขาย (Dashboard)

---

## 🌟 คุณสมบัติเด่น (Features)
- 🛒 **หน้าชำระเงิน (POS Interface)**:
  - แยกหมวดหมู่สินค้า กาแฟ และ เบเกอรี่ พร้อมระบบค้นหา
  - ตะกร้าสินค้าสามารถปรับเพิ่ม/ลดปริมาณ หรือลบสินค้าได้สะดวก
  - ระบบสะสมแต้มสมาชิก (คำนวณแต้มได้รับอัตโนมัติตามประเภทสินค้า)
  - ระบบแลกส่วนลดด้วยแต้มสะสม (แลก 10 แต้ม เพื่อเป็นส่วนลด 50 บาท)
  - ระบบออกใบเสร็จรับเงินแบบย่อ (Receipt ticket modal)
- 📊 **หน้าแดชบอร์ดสรุปผล (Sales Dashboard)**:
  - การ์ดตัวชี้วัดหลัก: รายได้วันนี้, จำนวนบิลคำสั่งซื้อทั้งหมด, แต้มที่แจกสะสม, จำนวนสมาชิกร้าน
  - กราฟแนวโน้มยอดขายรายวันแบบตอบสนอง (Interactive line chart ด้วย Chart.js)
  - สถิติอันดับสินค้าขายดีที่สุด 5 อันดับแรก
  - ประวัติรายการคำสั่งซื้อ 10 รายการล่าสุด พร้อมรายละเอียดใบเสร็จย้อนหลัง
- 👥 **หน้าจัดการลูกค้าสมาชิก (Loyalty Members)**:
  - ลงทะเบียนสมัครสมาชิกใหม่ด้วย ชื่อ และ เบเกอร์โทรศัพท์
  - ค้นหาสมาชิกในหน้าชำระเงินทันทีด้วยชื่อหรือเบอร์โทรศัพท์
  - ตารางรายชื่อสมาชิกพร้อมยอดแต้มสะสมปัจจุบัน
- ⚙️ **หน้าตั้งค่าฐานข้อมูล (Supabase Integration)**:
  - รองรับการเชื่อมต่อกับ Supabase Database ทั่วโลก
  - มีระบบสลับไปใช้โหมดออฟไลน์ทำงานบนบราวเซอร์ (Offline/Mock Mode) เมื่อไม่มีเครือข่ายหรือไม่ได้ตั้งค่าคีย์

```text

│   └── bakery_cake.jpg

├── styles.css           # สไตล์สไตล์ครีมพรีเมียม (Premium Light Mode CSS)
├── app.js               # ควบคุมการทำงานของระบบ POS, แดชบอร์ด และหน้าจอสมาชิก

└── setup.sql            # สคริปต์ SQL สำหรับนำไปรันใน Supabase Editor เพื่อสร้างตาราง
```

## 🚀 วิธีการติดตั้งและอัปโหลดขึ้น GitHub & Vercel

### ขั้นตอนที่ 1: อัปโหลดโค้ดขึ้น GitHub
1. สร้าง **Repository** ใหม่บนบัญชี GitHub ของคุณ (เช่น ตั้งชื่อว่า `espresso-pos`)
2. เปิด Terminal (PowerShell หรือ Command Prompt) ในเครื่องของคุณแล้วพิมพ์คำสั่งดังนี้:
   ```bash
   git init
   git add .
   git commit -m "Initial commit: EspressoPOS release"
   git branch -M main
   git remote add origin https://github.com/USERNAME/REPOSITORY-NAME.git
   git push -u origin main
   ```

### ขั้นตอนที่ 2: ตั้งค่าฐานข้อมูลใน Supabase
1. เข้าไปที่เว็บ [Supabase](https://supabase.com) และสร้างโครงการใหม่ (New Project)
2. เมื่อโครงการพร้อมใช้งานแล้ว ไปที่เมนู **SQL Editor** จากแถบเครื่องมือซ้ายมือ
3. กดปุ่ม **"New query"** เพื่อสร้างหน้าเขียน SQL ใหม่
4. เปิดไฟล์ `setup.sql` ในเครื่องของคุณ ก๊อปปี้คำสั่งทั้งหมดมาวางลงใน Editor แล้วกดปุ่ม **"Run"** ด้านขวาล่าง
5. ตาราง `customers`, `orders` และ `order_items` จะถูกสร้างขึ้นมาทันที

*คำแนะนำเพิ่มเติม: ในการสร้างรหัสผ่านพนักงานขายสำหรับเข้าสู่ระบบ ให้ไปที่ **Authentication -> Users -> Add User** เพื่อลงทะเบียนบัญชีสำหรับพนักงานล็อกอินเข้าร้าน*

### ขั้นตอนที่ 3: เผยแพร่เว็บลงบน Vercel (Deployment)
1. เข้าไปที่เว็บ [Vercel](https://vercel.com) และล็อกอินผ่านบัญชี GitHub ของคุณ
2. กดปุ่ม **"Add New" -> "Project"**
3. ค้นหาคลังเก็บโค้ด `espresso-pos` ที่เราได้อัปโหลดไว้บน GitHub แล้วกดปุ่ม **"Import"**
4. ในหน้าจอตั้งค่าก่อน Deploy ให้ไปที่ส่วนของ **Environment Variables** แล้วระบุคีย์ 2 ค่าดังนี้:
   - **`SUPABASE_URL`**: ระบุ URL ของโครงการ Supabase ของคุณ (หาได้จาก Settings -> API -> Project URL)
   - **`SUPABASE_ANON_KEY`**: ระบุ Anon Key ของโครงการ Supabase (หาได้จาก Settings -> API -> Project API Keys)
5. กดปุ่ม **"Deploy"** และรอประมาณ 1 นาที เว็บของคุณจะพร้อมใช้งานผ่าน URL สาธารณะจาก Vercel ทันที!
6. ระบบจะดึงคีย์มาเชื่อมต่อฐานข้อมูลโดยอัตโนมัติ ทำให้พนักงานล็อกอินใช้งานได้จากทุกอุปกรณ์อย่างปลอดภัย

## 💻 การรันโปรเจกต์และทดสอบในเครื่องส่วนบุคคล (Local Testing)
คุณไม่จำเป็นต้องมี Vercel หรือเซิร์ฟเวอร์ในการทดสอบระบบแบบ Local:
- เพียงเข้าไปในโฟลเดอร์โครงการ และดับเบิลคลิกไฟล์ `index.html` เพื่อเปิดใช้งานได้ทันที
- บราวเซอร์จะแสดงกล่องแจ้งเตือนสีเหลืองด้านบนว่ากำลังทำงานใน "Offline Mode" ซึ่งจะเก็บข้อมูลทั้งหมดของคุณไว้ในพื้นที่เก็บข้อมูลจำลอง (`localStorage`) ของบราวเซอร์เอง ทำให้คุณยังสามารถทดสอบ ฟังก์ชันขายสินค้า, การเพิ่มสมาชิก, การสะสมแต้ม และแดชบอร์ดสรุปสถิติได้แบบออฟไลน์ 100%
